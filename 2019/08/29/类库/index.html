委托

委托的主要好处是可以把方法作为参数/属性来赋值或传递。事件是委托的一种具体应用。委托可以理解为一种接口，具体导方法实现这个接口，在开发阶段只需要关系委托的定义就可以调用，而不用关心他如何实现的或者在哪里实现的。

在用线程的时候直接调用方法或者改变控件的话会出错，但是用委托就可以解决错误，可以实现异步更新。

举例：委托应用于“同步调用”的场景，入一个有声阅读器，打开记事本（1）并开始阅读（2），这是两个方法，但这时，我想调用打开的方法时，同时调用阅读，这时候委托就来了，把阅读这个方法作为参数传给“打开笔记本” 的方法。

委托就是对一个方法映射，把方法名当参数来传递，类似于JS中的在某个function的参数中把另一个function的名称当参数来传递

类库

类库（class library）是一个综合性的面向对象的科重用类型集合，这些类型包括：接口、抽象类、具体类。

类库可以解决一系列厂家编程任务（包括诸如字符串管理、数据收集、数据库链接以及文件访问等任务）还可以解决多种专用开发任务（控制台应用程序、桌面应用程序、WEB用用程序）。

类库字面意思就是类的集合，类库不等于框架。Framework是比类库更高一级的概念。

类库会在编译之后生成一个dll文件，然后其他的项目可以引用这个.dll(可扩展)

多线程

一、进程、线程及多线程的概

进程：进程就是一个应用程序开始运行，那么这个应用程序就会存在一个属于这个应用程序的进程。

线程：就是进程中的基本执行单元，每个进程中都至少存在着一个线程，这个线程是根据进程创建而创建的，所以这个线程我们称之为主线程。那么多线程就是包含有除了主线程之外的其他线程。如果一个线程可以执行一个任务，那么多线程就是可以同时执行多个任务。

二、线程的基本知识

Thread 类

Thread 类是用于控制线程的基础类，它存在于 System.Threading 命名空间。通过 Thread 可以控制当前应用程序域中线程的创建、挂起、停止、销毁。

Thread一些常用属性：

ExccutionContext

属性名称 说明
ExecutionContext 获取一个ExecutionContext对象，该对象包含有当前进程的各种上下文信息
Name 获取或设置线程的名称（只能设置一次）
IsAlive 获取一个值，该值指示当前线程的执行状态
IsBackground 获取或设置一个值，该值指示某个线程是否为后台线程
IsThreadPoolThread 获取 一个值，该值指示线程是否属于托管线程池。
ManagedThreadId 获取当前托管线程的唯一标识符
CurrentThread 获取当前正在运行的线程
CurrentContext 获取线程正在其中执行的当前上下文。
ThreadState 获取一个值，该值包含当前线程的状态
Priority 获取或设置一个值，该值指示线程的调度优先级

Thread一些常用方法

方法名称 说明
Interrupt() 用于中断处于WaitSleepJoin(即阻塞或等待的线程状态)线程状态的线程。
About() 用于终止当前线程
Join() 阻塞调用线程，直到某个线程终止为止
MemoryBarrier() 取消为当前线程请求的About(),即取消终止当前线程的操作
Sleep() 将当前线程挂起或阻塞指定的时间
Start() 启动当前线程
Resume() 继续已挂起的线程
Suspend() 挂起线程，或者如果线程已挂起，则不起作用
